# Нагрузочное тестирование с Locust

Для запуска нагрузочных тестов на приложении FLAI:

## Предварительные требования

- Python 3.9 или выше
- Приложение FLAI должно быть запущено (например, через `docker-compose up`)

## Настройка (виртуальное окружение)

Чтобы избежать ошибки "externally-managed-environment", возникающей в современных дистрибутивах Linux, настоятельно рекомендуется использовать виртуальное окружение Python.

### Пошаговая настройка

1. **Перейдите в корневую директорию проекта** (где находится `docker-compose.yml`).

2. **Создайте виртуальное окружение**:
```bash
python3 -m venv venv
```
3. **Активируйте виртуальное окружение**:
    - На Linux/macOS:
    ```bash
    source venv/bin/activate
    ```
    - На Windows:
    ```bash
    venv\Scripts\activate
    ```
4. **Установите Locust** (и любые другие необходимые пакеты):
```bash
pip install locust
```
5. **Запустите Locust с предоставленным тестовым скриптом**:
```bash
locust -f tests/load/locustfile.py --host http://localhost:5000
```
6. **Откройте веб-интерфейс Locust** по адресу `http://localhost:8089` и запустите тест.

## Альтернатива: использование pipx
Если вы не хотите каждый раз активировать виртуальное окружение, можно установить Locust через pipx:
```bash
sudo apt install pipx
pipx ensurepath
# Перезапустите терминал или выполните source ~/.bashrc
pipx install locust
```
Затем запускайте Locust из любого места:
```bash
locust -f tests/load/locustfile.py --host http://localhost:5000
```

## Запуск без веб-интерфейса (headless)
Для автоматического тестирования можно запустить Locust без веб-интерфейса:
```bash
locust -f tests/load/locustfile.py --host http://localhost:5000 --headless -u 10 -r 2 --run-time 1m
```
- `-u` : количество пользователей
- `-r` : скорость появления пользователей (пользователей в секунду)
- `--run-time` : продолжительность теста

## Тестовый пользователь
Тестовый скрипт использует пользователя `testuser` с паролем `testpass`. Этот пользователь должен существовать в системе FLAI перед запуском теста. Вы можете создать его через панель администратора (войдите как администратор) или через API.

## Примечания
Тест создаёт новую сессию для каждого симулированного пользователя и использует её для последующих запросов.
Тест предполагает, что приложение работает по адресу `http://localhost:5000`. При необходимости измените параметр `--host`.
Результаты отображаются в веб-интерфейсе Locust или выводятся в консоль при запуске в режиме headless.
